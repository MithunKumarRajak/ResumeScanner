Implementation Blueprint — file-by-file
-------------------------------------

Goal: produce a minimal, runnable V4 implementation that extends the existing project with
multi-use resume capabilities (classification, matching, skill extraction, quality scoring).

How to use this blueprint

- Follow the file map below to add/modify files in the workspace.
- Run the training script to produce artifacts in `FullStackApp/v4/`.
- Start the FastAPI server in `FullStackApp/backend/` to serve endpoints.

1) New/Updated project files

- `FullStackApp/backend/app/api.py` — API router for v4 endpoints
  - Endpoints: `/v4/predict-category`, `/v4/match`, `/v4/extract-skills`, `/v4/quality-score`
  - Loads artifacts from `FullStackApp/v4/` on startup and applies the same preprocessing as training.

- `FullStackApp/backend/app/models_v4.py` — lightweight wrappers for loading `model.pkl`,
  `tfidf.pkl`, `encoder.pkl`, and an `embedder` (optional) for semantic matching.

- `FullStackApp/backend/app/preprocess.py` — canonical `clean_text` and `spacy_preprocess`
  functions (copy from `ResumeModel_v3.py`) packaged so API and training share identical logic.

- `FullStackApp/train/train_v4.py` — training pipeline that:
  - Loads and unifies datasets (same as v3 but writes a `manifest.json` for artifacts)
  - Builds: TF-IDF vectorizer (or optional transformer encoder), linear classifier baseline,
    dual-encoder training stub for resume-job matching, and skill-extraction training stub.
  - Saves artifacts to `FullStackApp/v4/` (model.pkl, tfidf.pkl, encoder.pkl, manifest.json).

- `FullStackApp/v4/manifest.json` — generated by training script; contains metadata:
  - dataset_hash, model_version, training_datetime, metrics, artifact_paths.

- `FullStackApp/backend/requirements.txt` — ensure server dependencies (FastAPI, uvicorn, joblib,
  scikit-learn, spacy, pandas) and pinned versions.

1) Minimal API behavior (design notes)

- `/v4/predict-category` (POST) accepts `{"resume_text": "..."}` and returns:
  - `category`, `confidence`, `probabilities` (top-5), `model_version`.

- `/v4/match` (POST) accepts `{"resume_text":"...", "job_description":"..."}` and returns:
  - `score` (0-1), `confidence`, `top_skills_overlap`, `explanation`.

- `/v4/extract-skills` (POST) accepts `{"resume_text":"..."}` and returns normalized skill list
  and mapping to canonical ontology IDs (if present).

- `/v4/quality-score` (POST) returns `score` (0-1) and `reasons` (readability, keyword_coverage,
  experience_signal).

1) Example minimal code snippets (adapt into files above)

- `preprocess.py` (core functions — copy/paste from v3):

```python
import re
import spacy

nlp = spacy.load('en_core_web_sm')

def clean_text(text: str) -> str:
    text = re.sub(r'http\S+|www\S+|https\S+', ' ', text)
    text = re.sub(r'[^a-zA-Z\s]', ' ', text)
    return re.sub(r'\s+', ' ', text).strip()

def spacy_preprocess(text: str) -> str:
    doc = nlp(text)
    tokens = [t.lemma_.lower() for t in doc if not t.is_stop and not t.is_punct and len(t.text)>1]
    return ' '.join(tokens)
```
